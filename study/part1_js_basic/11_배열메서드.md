# 자바스크립트에서 말하는 배열이란?

## 일반적인 배열

- **밀집 배열** (dense array):

  - 배열 요소들의 타입이 같으며
  - 동일한 메모리 공간
  - 연속적으로 나열되어 있어야

<br/>

## 자바스크립트 배열

- **희소 배열** (sparse array):

  - 요소들의 타입이 다를 수 있음
  - 배열 요소의 메모리 공간이 동일한 크기를 갖지 않아도됨
  - 연속적으로 나열되어 있지 않을 수 있음

- 자바스크립트 배열은 일반적으로 알고 있는 배열의 동작을 해시테이블로 구현한 **객체**다.
- 인덱스와 length를 프로퍼티 키로 갖고, 요소는 인덱스 키의 벨류이다.

  ```js
  const arr = [1, 2, 3];
  console.log(arr);
  // {0:1, 1:2, 2:3, length:3}
  ```

- 참고로, 희소 배열의 length 프로퍼티 값의 요소의 개수와 다르다. 즉, 비어 있어도 length에 count된다.
- 이처럼 자바스크립트는 문법적으로 희소 배열을 허용하지만 되도록이면 희소 배열을 만들지 않는 것이 좋다.
- delete 연산자를 권장하지 않는 이유도 여기에 있다. 이와 관련해서는 '요소의 삭제'에서 더 자세히 살펴보자.

<br/>

## 일반적인 배열 vs 자바스크립트 배열

이러한 구조적 차이점이 의미하는 것은 무엇일까?
알고리즘과 시간 복잡도에 대해 여기서 자세히 살펴볼 수는 없지만 대략적으로 살펴보자.

- 일반적인 배열

  - 접근: 인덱스로 바로 접근(임의 접근, random access) -> 시간 복잡도 O(1)
  - 삽입 & 삭제: 요소를 연속적으로 유지하기 위해 요소를 이동시켜야하는 비효율성 존재

- 자바스크립트 배열

  - 접근: 사실 해시 테이블로 구현된 객체이므로 접근이 일반적인 배열보다 느림
  - 삽입 & 삭제: 일반적인 배열보다 빠름

- 따라서 자바스크립트의 배열은 상대적으로 삽입, 삭제에 최적화된 자료구조라고 할 수 있다.
